# 配置和依赖说明

本文档汇总 CollabPlatform 运行所需的**环境要求、依赖清单、环境变量、数据库与可选服务配置**，作为部署的统一参考。内容以仓库中实际的 `package.json`、`prisma/schema.prisma`、`config/env.js` 与 `vite.config.js` 为准。

---

## 一、环境要求

| 组件 | 版本要求 | 说明 |
|------|----------|------|
| **Node.js** | **≥ 18** | 后端 AI 助手依赖全局 `fetch`（Node 18 起内置），低于 18 会报 `fetch is not defined` |
| **npm** | ≥ 8（随 Node 附带） | 前后端均用 npm 管理依赖 |
| **PostgreSQL** | 14+（推荐 Supabase 托管） | 主数据库，通过 Prisma 访问 |
| **Redis** | 6+（**可选**） | 未配置时自动降级为无缓存模式，不影响启动 |
| 操作系统 | Windows / macOS / Linux | 开发在 Windows 11 + PowerShell 环境验证过 |

> 前后端分离：前端默认 `localhost:5173`（Vite），后端默认 `localhost:3000`（Express）。

## 五、环境变量配置

### 5.1 后端 `server/.env`

由 `server/src/config/env.js` 加载并校验，**缺少 `DATABASE_URL` 或 `JWT_SECRET` 会直接退出进程**。

| 变量 | 必填 | 默认值 | 说明 |
|------|:----:|--------|------|
| `DATABASE_URL` | ✅ | — | PostgreSQL 连接串。Supabase 用**连接池**地址（端口 **6543**），运行时查询走此地址 |
| `DIRECT_URL` | 迁移时需要 | — | Supabase **直连**地址（端口 **5432**），供 Prisma 迁移 / `db push` 建表 |
| `JWT_SECRET` | ✅ | — | JWT 签名密钥，填任意随机字符串 |
| `JWT_EXPIRES_IN` | ✕ | `15m` | access token 有效期（开发期可设 `7d` 减少重登） |
| `JWT_REFRESH_EXPIRES_IN` | ✕ | `7d` | refresh token 有效期 |
| `PORT` | ✕ | `3000` | 后端监听端口 |
| `NODE_ENV` | ✕ | `development` | 运行模式（影响错误堆栈打印等） |
| `CLIENT_ORIGIN` | ✕ | `http://localhost:5173` | CORS 放行的前端来源 |
| `REDIS_URL` | ✕ | 空 | Redis 连接串；留空则以无缓存模式运行 |
| `DEEPSEEK_API_KEY` | ✕ | 空 | AI 助手密钥，留空则 AI 功能不可用（其余功能正常） |
| `DEEPSEEK_BASE_URL` | ✕ | `https://api.deepseek.com` | AI 服务地址（OpenAI 兼容接口） |
| `DEEPSEEK_MODEL` | ✕ | `deepseek-chat` | 使用的模型名 |

**`server/.env` 示例：**

```env
DATABASE_URL=postgresql://postgres.项目ID:密码@aws-1-ap-northeast-1.pooler.supabase.com:6543/postgres
DIRECT_URL=postgresql://postgres:密码@db.项目ID.supabase.co:5432/postgres
JWT_SECRET=请替换为随机字符串
JWT_EXPIRES_IN=7d
JWT_REFRESH_EXPIRES_IN=7d
CLIENT_ORIGIN=http://localhost:5173
# 可选
REDIS_URL=
DEEPSEEK_API_KEY=
```

### 5.2 前端 `client/.env`

Vite 只暴露以 `VITE_` 开头的变量（通过 `import.meta.env` 读取）。

| 变量 | 必填 | 默认回退 | 说明 |
|------|:----:|----------|------|
| `VITE_WS_URL` | ✅ | `window.location.origin` | Socket.io 服务端地址。**必须显式设置**，否则会回退到 `:5173`，而 Vite 只代理 `/api` 不代理 WebSocket，导致协作通道连不上 |
| `VITE_API_BASE_URL` | ✕ | `/api` | REST 接口前缀。留空时走 Vite 的 `/api` 代理转发到后端；也可填完整地址如 `http://localhost:3000/api` |

**`client/.env` 示例：**

```env
VITE_API_BASE_URL=http://localhost:3000/api
VITE_WS_URL=http://localhost:3000
```

## 六、数据库配置（Prisma + PostgreSQL）

- **Provider**：`postgresql`，连接 `DATABASE_URL`，迁移用 `DIRECT_URL`（见 `prisma/schema.prisma`）。
- **数据模型**：共 6 张表，主键统一 UUID：
  `User`、`CollaborativeItem`（type: Whiteboard / Document）、`Permission`（owner/editor/viewer）、`Version`、`Folder`（自关联树）、`Comment`（`parent_id` 自关联回复）。
- **级联规则**：删 `CollaborativeItem` → 级联删 `Version`/`Comment`/`Permission`；删 `Folder` → 子文件夹级联删、其下文档 `folder_id` 置空（`SetNull`，文档退回未分类）。
- **迁移记录**（`prisma/migrations/`）：`init` → `add_comment_parent_id` → `add_folder_tree`。

**初始化数据库（二选一）：**

```bash
cd server

# 方式一：Prisma（推荐，需 DIRECT_URL / 5432 端口可达）
npx prisma db push        # 将 schema 推送到数据库建表
npx prisma generate       # 生成 Prisma Client

# 方式二：5432 不可达时，在 Supabase SQL Editor 手动执行建表语句，
#         再本地运行 npx prisma generate
```

## 七、可选服务配置

### 7.1 Redis（缓存，可选）

- 由 `server/src/config/redis.js` 用 ioredis 连接。
- `REDIS_URL` 为空或连接失败时，**打印警告但不崩溃**，导出一个空操作（no-op）Proxy 客户端，业务代码无需关心 Redis 是否可用。
- 需要启用时填 `REDIS_URL=redis://localhost:6379` 之类的连接串。

### 7.2 AI 助手（DeepSeek，可选）

- 在 `server/.env` 填入 `DEEPSEEK_API_KEY` 后即启用；未填时 AI 相关接口不可用，其余功能不受影响。
- 通过 OpenAI 兼容的 `/chat/completions` 接口，以 **SSE 流式**逐字返回。
- 可选覆盖 `DEEPSEEK_BASE_URL`、`DEEPSEEK_MODEL`。
- **依赖 Node 18+ 的全局 `fetch`**，未引入额外依赖。

## 八、端口与网络

| 服务 | 默认端口 | 可配置项 |
|------|----------|----------|
| 前端 Vite Dev Server | 5173 | `client/vite.config.js` 的 `server.port` |
| 后端 Express + Socket.io | 3000 | `server/.env` 的 `PORT` |
| PostgreSQL（Supabase 池） | 6543 | `DATABASE_URL` |
| PostgreSQL（Supabase 直连） | 5432 | `DIRECT_URL` |
| Redis | 6379 | `REDIS_URL` |

- Vite 开发代理：`/api` → `http://localhost:3000`（见 `vite.config.js`）。**WebSocket 不走此代理**，故 `VITE_WS_URL` 必须指向后端真实地址。
- 后端 CORS 由 `CLIENT_ORIGIN` 放行前端来源。

## 九、完整安装与启动

```bash
# 1. 安装依赖
cd server && npm install && cd ..
cd client && npm install && cd ..

# 2. 配置环境变量
#    - 编辑 server/.env（至少 DATABASE_URL、JWT_SECRET）
#    - 编辑 client/.env（至少 VITE_WS_URL）

# 3. 初始化数据库
cd server && npx prisma db push && npx prisma generate && cd ..

# 4. 启动（分两个终端）
cd server && npm run dev     # 后端 http://localhost:3000
cd client && npm run dev     # 前端 http://localhost:5173
```

**可用脚本：**

| 位置 | 命令 | 作用 |
|------|------|------|
| server | `npm run dev` | nodemon 热重载启动 |
| server | `npm start` | 生产方式启动（`node src/index.js`） |
| server | `npm run prisma:generate` | 生成 Prisma Client |
| server | `npm run prisma:migrate` | 执行迁移（`prisma migrate dev`） |
| client | `npm run dev` | Vite 开发服务器 |
| client | `npm run build` | 生产构建到 `dist/` |
| client | `npm run preview` | 预览构建产物 |

## 十、常见配置问题

| 现象 | 原因与解决 |
|------|-----------|
| 后端启动即退出，提示「缺少必要环境变量」 | `server/.env` 未设 `DATABASE_URL` 或 `JWT_SECRET` |
| `fetch is not defined`（AI 功能） | Node 版本低于 18，升级到 18+ |
| 编辑器白屏 / 报「找不到 @tiptap/...」 | 拉取文档模块改动后未在 `client` 重新 `npm install` |
| 多人编辑不同步 | `VITE_WS_URL` 未指向后端（回退到 :5173 导致 WS 连不上）；或用户未登录 / 权限不足（需 editor+） |
| `prisma db push` 报 `EPERM` | 后端服务未停止，Prisma Client 文件被占用 |
| 数据库连接失败 | 检查连接串格式，Supabase 密码需去掉方括号；5432 不可达时改用 SQL Editor 建表 |
| Redis 警告 | 正常，未配置时自动降级为无缓存模式，不影响运行 |
